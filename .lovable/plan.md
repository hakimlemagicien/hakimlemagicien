## المرحلة 1 — شاشة الدفع (Frontend فقط، بدون backend)

تظهر **فقط** لمن اختار التدريب أونلاين (بعد `PricingScreen`).

### المسار الجديد داخل الفانل
```
... → Pricing (اختيار الباقة) → Payment (جديدة) → PaymentProof (Modal)
```

سأضيف داخل `PricingScreen` تمرير الباقة المختارة (`selectedTier`) عند الضغط على "أريد هذه الباقة"، وأنتقل إلى `step = "payment"` بدلاً من السلوك الحالي.

### مكونات الشاشة (مطابقة للصورة المرجعية)
1. **Header**: زر "رجوع" + عنوان "إتمام الدفع" + شارة "دفع آمن ومشفر 100%".
2. **Summary Card**: اسم الباقة، السعر الكلي، المدة، أيقونة كأس SVG، علامة صح برتقالية.
3. **عنوان**: "اختر طريقة الدفع".
4. **4 بطاقات إلكترونية**: Binance Pay (موصى به + Badge "الأسرع")، PayPal، Skrill، Wise.
   - شعارات SVG حقيقية inline (ليست emojis).
   - عند الاختيار: تغيّر لون البطاقة + علامة صح + اهتزاز خفيف + animation.
5. **قسم التحويل البنكي**: 3 بطاقات (الإمارات/المغرب/البرازيل) — كل واحدة تفتح Modal فيه بيانات الحساب + زر نسخ.
6. **بطاقة WhatsApp مساعدة**.
7. **شريط الثقة السفلي**: 4 عناصر (تشفير، حماية، استرجاع، دعم) + سطر "SSL 256-bit".

### بعد اختيار وسيلة الدفع
يظهر زر برتقالي كبير عريض: **"لقد قمت بالدفع"** → يفتح Modal لرفع إثبات الدفع (image/PDF/screenshot) — يُخزَّن مؤقتاً في state فقط في هذه المرحلة.

### شريط التقدم
يُحدَّث ليصبح 14 خطوة للأونلاين (Payment = 14/14) و 15 للحضوري في دبي (إن لزم — لكن الحضوري لا يمر بهذه الشاشة، يبقى 14).

### الملفات
- `src/routes/quiz.tsx`: إضافة `"payment"` لـ `Step`، تمرير `selectedTier` من `PricingScreen`، مكوّن جديد `PaymentScreen` + `PaymentProofModal` + `BankDetailsModal`.

## المرحلة 2 — Backend والحساب (تتطلب تفعيل Lovable Cloud)

هذه المرحلة **لا يمكن تنفيذها بدون backend** لأنها تتضمن:
- إنشاء حساب مستخدم حقيقي
- تخزين كلمة المرور بشكل آمن
- تخزين بيانات الباقة والاشتراك
- تسجيل دخول لاحقاً
- صفحة Dashboard مرتبطة بمستخدم

### ما سيُبنى
1. تفعيل **Lovable Cloud** (Supabase تحت الغطاء).
2. جدول `profiles` (الاسم، الهاتف، المدينة، الهدف، نوع التدريب، الباقة، تاريخ الاشتراك).
3. جدول `payments` (الباقة، السعر، طريقة الدفع، إثبات الدفع، الحالة).
4. Storage bucket لإثباتات الدفع.
5. شاشات ما بعد الدفع:
   - **PaymentSuccess**: animation + confetti + أيقونة نجاح 3D + auto-redirect بعد 3 ثوانٍ.
   - **CreatePassword**: كلمة مرور + تأكيد → ينشئ الحساب بالبريد/الهاتف المخزّن مسبقاً.
   - **Welcome**: ترحيب بالاسم + ملخص الباقة + زر "الدخول إلى حسابي".
   - **Dashboard** (`/dashboard`): 6 أقسام (ترحيب، شريط تقدم اليوم 1/90، ملخص الحالة، الباقة، إشعارات، الفواتير).
   - **Login** (`/login`): بريد أو هاتف + كلمة مرور.
6. حماية `/dashboard` بـ `_authenticated` layout.
